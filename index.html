Jenkins Configuration Management with Ansible

A typical Jenkins setup

  • No test environments
    • Every change risks breaking the pipeline
    • Leads to lots of junk commits when used with VCS
  • Configured using the GUI
    • Includes irrelevant settings
    • No comments
    • No abstractions (or even copy & paste)
    • Unreadable/un-diffable XML configuration
  • Junk commits + generated configuration = maintenance hell
    • Hard to answer questions like: who changed what, when, how, and why?
    • Hard to find problematic changes and roll them back

Job DSL and Pipeline

  • Solve many problems by replacing GUI-based configuration with Groovy-based DSL scripts
    • Describe only the relevant parts of the configuration
    • Allow comments
    • Provide constructs for abstraction
    • Human-authored so easy to read and diff
  • Unfortunately ignore the idea of separate testing environments
