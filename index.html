Jenkins Configuration Management with Ansible

Timo Mihaljov / Solita

A typical Jenkins setup

  • GUI-based configuration
    • Irrelevant settings
    • No comments
    • No abstractions
    • Can't copy & paste
    • Generated XML
  • A single environment
    • Testing in production
    • Junk commits
  • Messy VCS history
    • Hard to tell who, what, when, how, and why?
    • Hard to roll back individual changes

Job DSL and Pipeline

  • Groovy-based DSL scripts
    • Only relevant configuration
    • Comments
    • Constructs for abstraction
    • Human-authored
  • A single environment
    • Testing in production
    • Junk commits

Ansible to the rescue!

  • Multiple environments
    • Inventory per env
    • Group vars for env details
  • Templating for DSL scripts
  • Develop in Vagrant
    • Test before commit
    • Commit before prod

Environments

  • dev
    • Jenkins, config from workdir
    • Defined in Vagrantfile
  • local
    • Deployment target for dev Jenkins
    • Defined in environments/local/
  • ci
    • Jenkins, config from VCS
    • Defined in environments/ci/
  • qa & prod
    • Deployment targets for ci Jenkins
    • Defined in environments/<env>/

Dev Environment

Vagrantfile

config.vm.synced_folder '../hello-java', '/hello-java'
config.vm.provision 'ansible' do |ansible|
  ansible.playbook = 'site.yml'
  ansible.groups = {
    'jenkins' => ['default'],
    'jenkins:vars' => {'app_path'       => '/hello-java',
                       'ansible_path'   => '/vagrant',
                       'qa_inventory'   => 'environments/local/inventory',
                       'prod_inventory' => 'environments/local/inventory'},
    'app' => ['default']
  }
end

Server Configuration

site.yml

- hosts: jenkins
  vars:
    solita_jenkins_security_realm: none
    solita_jenkins_plugins:
      - blueocean
      - workflow-aggregator
      - ansible
  roles:
    - solita.jenkins
    - solita.ansible

Job Config

jobs/Hello.groovy.j2

APP_GIT = "{{ app_git | default('') }}"
pipelineJob('hello') {
    scm {
        if (APP_GIT) {
            git(APP_GIT)
        }
    }
    triggers {
        scm('H/5 * * * *')
    }
    definition {
        cps {
            script(readFileFromWorkspace('pipeline/Hello.groovy'))
            sandbox()
        }
    }
}

Pipeline Config – Checkout

jobs/pipeline/Hello.groovy.j2

node {
    stage('Checkout') {
        if (APP_GIT) {
            checkout([
                $class: 'GitSCM',
                extensions: [[
                    $class: 'RelativeTargetDirectory',
                    relativeTargetDir: 'app'
                ]],
                userRemoteConfigs: [[url: APP_GIT]]
            ])
        } else if (APP_PATH) {
            // Ignore "permission denied" errors
            sh "rsync -av ${APP_PATH}/ app | true"
        }
    }
